The following ultron components are clean-room re-implementations whose
*approach* is informed by the corresponding OpenClaw core modules.
Algorithm shapes and contract names are adapted; no source code is
copied verbatim. The OpenClaw source was read under the binding ClawHub
batch security rules (read-only, quarantined, pattern-level extraction
only) and the per-technique tier annotations (GREEN / YELLOW / RED) are
recorded against the safety stack each component participates in.
Ultron's versions are restructured to fit the voice-first, single-host,
single-user, native-Windows + Python runtime; OpenClaw is a multi-tenant
TypeScript gateway with plugin marketplace, so most concrete code shape
differs while the pattern semantics are preserved.
